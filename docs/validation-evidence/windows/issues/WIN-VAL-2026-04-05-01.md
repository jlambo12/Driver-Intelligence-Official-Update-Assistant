# WIN-VAL-2026-04-05-01 — WMI ACL runbook hardening

- Date opened: 2026-04-05
- Date updated: 2026-04-05
- Owner: Platform QA
- Severity: Medium
- Source evidence: EV-002 (restricted WMI validation)
- Status: ✅ Closed

## Problem

В restricted WMI окружениях оператору требовался более явный runbook по диагностике namespace ACL, чтобы быстрее отличать policy-driven access denials от реальной поломки runtime.

## Resolution

Закрыто за счёт добавления отдельного runbook:
- `docs/runbooks/Restricted-WMI-ACL-Runbook.md`

В runbook добавлены:
1. expected degraded-path поведение (`Partial/Failed` без crash);
2. операторский quick-check чеклист;
3. support handoff checklist;
4. привязка в release checklist (`docs/Release-GoNoGo-Checklist.md`).

## Exit criteria

- [x] Runbook добавлен и утверждён QA + Runtime owner.
- [x] Повторный restricted-WMI прогон выполняется без эскалации как инцидент.
